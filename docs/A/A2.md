# 文章與章節

## 基本概念

這個網站有兩個東西要分清楚：

```text
1. 文章檔案
   實際存在 docs 裡面的 .md 檔案

2. 左側目錄
   顯示在網站左邊的書本目錄
```

## 左側目錄是什麼？

左側目錄就像一本書的目錄，有許多章節。

例如網站左邊看到：
```text
開始
├─ 前置作業
└─ 文章操作

數學互動範例
└─ 著色問題
```

對應到設定檔會長這樣：

```ts
sidebar: [
  {
    text: '開始',
    collapsed: false,
    items: [
      { text: '前置作業', link: '/A/A1' },
      { text: '文章操作', link: '/A/A2' }
    ]
  },
  {
    text: '數學互動範例',
    collapsed: false,
    items: [
      { text: '著色問題', link: '/graph-coloring/graph-coloring' }
    ]
  }
]
```

## 名稱說明

```text
章節：左側目錄的大標題
文章：章節底下可以點的項目
檔案：實際存在 docs 裡的 .md 檔案
```

例如：

```text
開始              ← 章節
前置作業          ← 文章
docs/A/A1.md      ← 檔案
```

## sidebar 裡每個欄位的意思

```ts
{
  text: '開始',
  collapsed: false,
  items: [
    { text: '前置作業', link: '/A/A1' }
  ]
}
```

說明：

```text
text       顯示名稱
collapsed  是否預設收合
items      這個章節底下有哪些文章
link       點下去要打開哪一篇文章
```

### collapsed 是什麼？

```ts
collapsed: false
```

代表預設展開。

```ts
collapsed: true
```

代表預設收合。

## 檔案和 link 的對應規則

最重要規則：

```text
docs 後面的路徑，就是 link
但是不用寫 .md
```

範例：

| 實際檔案位置 | sidebar 裡的 link |
|---|---|
| `docs/index.md` | `/` |
| `docs/A/A1.md` | `/A/A1` |
| `docs/A/A2.md` | `/A/A2` |
| `docs/graph-coloring/graph-coloring.md` | `/graph-coloring/graph-coloring` |

例如：

```text
docs/A/A1.md
```

對應：

```ts
link: '/A/A1'
```
